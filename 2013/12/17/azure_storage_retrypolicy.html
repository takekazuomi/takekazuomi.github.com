Windiows Azure Storage Client の RetryPolicy

Windiows Azure Storage では、動的な負荷分散（Load Balancing）と障害時の自動フェールオーバーが実装されています。そのためアプリケーションを安定して動作させるにはリトライ処理が欠かせません。 日本語だと、 Windows Azure ストレージ サービスの構造とリトライ ポリシーの関係について 英語だと Storage Client Library 2.0 – Implementing Retry Policies や、 Windows Azure Storage Client Library 2.0 Breaking Changes & Migration Guide 、 最新のWindiows Azure Storage Client 3.0 で導入された ExtendedRetryPolicy については、 Windows Azure Storage Redundancy Options and Read Access Geo Redundant Storage に記述されています。

少々情報が散逸していて分かりづらいので最新の Windiows Azure Storage Client 3.0 のコードを確認して調べた範囲をまとめます。
